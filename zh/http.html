概述

强大的异步HTTP服务器和客户端，支持WebSocket。

简洁和更有效地替代Spring、Jetty、Vert.x和其他类似解决方案。 ActiveJ HTTP的性能明显优于它们，这一点已被许多 基准测试所证明。

提供工具，以简单和方便的方式建立具有异步I/O的HTTP服务器和客户端。

public final class HttpHelloWorldExample extends HttpServerLauncher {  @Provides  AsyncServlet servlet() {    return request -> HttpResponse.ok200().withPlainText("Hello World");  }
  public static void main(String[] args) throws Exception {    Launcher launcher = new HttpHelloWorldExample();    launcher.launch(args);  }}

特点​

  • 完全异步
  • 没有第三方依赖，建立在 ActiveJ Net 模块之上。
  • 没有遗留的适配器层
  • 支持WebSocket
  • 支持TLS
  • 可以作为一个应用程序的Web服务器：支持外部提供的 ActiveJ Inject (DI) 模块的业务逻辑和 AsyncServlets
  • 正如 基准测试所证明的那样，速度极快
  • 高度优化和GC友好。
    • 自动回收 ByteBufs 与 HttpRequest/HttpResponse 以及 ByteBufs 从 Net async sockets收到的字节库。
    • 优化的标题多图和内部URL表示，垃圾对象产量低
    • 专门的头文件子类将其内容直接渲染到 ByteBuf，没有中间的垃圾对象。

敏捷的Servlet​

  • 基本上，它只是一个异步函数，将 HttpRequest 映射到 HttpResponse
  • 预定义的 AsyncServlet 出箱子(StaticServlet, BlockingServlet RoutingServlet
  • 广泛支持 AsyncServlets的功能组成。 RoutingServlet 用于建立servlet路由。
    • 灵活地将HTTP路径和方法映射到 AsyncServlets （包括其他RoutingServlets）。
    • 支持路径参数（如 /users/:id）和相对路径

在你的项目中加入ActiveJ HTTP​

您可以通过导入ActiveJ HTTP的Maven资源库将其添加到您的项目中。 这些文档涵盖了最新发布的ActiveJ HTTP v5.1 （见上 Github 或 Maven）。)
